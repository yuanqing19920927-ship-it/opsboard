## V1 — 基础监控平台（已完成）

核心监控能力搭建，覆盖服务器、数据库、端口、资产四大板块。

| 功能 | 说明 | 状态 |
|------|------|------|
| Agent 指标采集 | Go Agent 通过 gRPC 上报 CPU/内存/磁盘/网络/容器/GPU 指标 | ✅ |
| WebSocket 实时推送 | 全局单连接，服务器指标实时更新到前端 | ✅ |
| VictoriaMetrics 存储 | 所有指标写入 VM，支持 PromQL 查询 | ✅ |
| 仪表盘 | 全局概览：服务器状态列表 + 端口摘要 + 资源排行 + 统计卡片 | ✅ |
| 服务器列表 | 卡片/表格双视图，实时 CPU/内存/磁盘进度条 | ✅ |
| 服务器详情 | 基本信息 + 实时概览（含 GPU）+ Docker 容器表格 + 运行业务 | ✅ |
| 端口监控 | TCP 端口探测规则 CRUD + 实时状态卡片 + 自动刷新 | ✅ |
| 托管业务 | 按服务器分组的资产表格，CRUD + 技术栈标签 | ✅ |
| 数据库 RDS 监控 | 阿里云 RDS 实例列表 + 实时指标瓦片 | ✅ |
| 阿里云 ECS 集成 | 通过 CloudMonitor API 采集 ECS 实例指标 | ✅ |
| Nginx 统一入口 | 反代 HTTP/WebSocket/VictoriaMetrics | ✅ |

## V5 — 网络拓扑探知（已完成）

Server 集中扫描，发现和管理内网网络设备。

### 5.1 网络设备发现

| 功能 | 说明 | 状态 |
|------|------|------|
| ICMP 扫描 | 跨网段 ping sweep，发现活跃设备（速率限制 10 并发/10ms 间隔） | ✅ |
| OUI 厂商识别 | MAC 前缀查询 80+ 厂商（Cisco/Huawei/H3C/Aruba 等） | ✅ |
| SNMP 自动探测 | v2c 探测 UDP 161，读取 sysDescr/sysName，推断设备类型 | ✅ |
| LLDP/CDP 邻居发现 | 通过 SNMP GET 被动读取邻居表，构建拓扑链路 | ✅ |
| 安全约束 | hard cap 速率限制、/24 最小前缀、禁止广播包、单网段顺序扫描 | ✅ |
| SNMP 凭据加密 | community string 通过 credentials 系统加密存储 | ✅ |

### 5.2 拓扑可视化

| 功能 | 说明 | 状态 |
|------|------|------|
| D3.js 力导向图 | 设备节点 + LLDP/CDP 链路，按类型图标区分 | ✅ |
| 设备列表 | 表格视图，筛选/搜索/内联编辑设备类型 | ✅ |
| 网段概览 | 按子网自动分组，卡片展示在线率 | ✅ |
| 扫描管理 | 二次确认弹窗 + WebSocket 进度 + 取消支持 | ✅ |

### 5.3 连通性监控

| 功能 | 说明 | 状态 |
|------|------|------|
| 定时 ping | 每 60 秒检测所有已发现设备 | ✅ |
| 离线检测 | 连续 3 次失败标记 offline，1 次成功恢复 online | ✅ |
| 告警集成 | `network_device_offline` 告警类型，复用 AlertEngine | ✅ |
| WebSocket 推送 | 设备状态变化实时推送前端 | ✅ |
